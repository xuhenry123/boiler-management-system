#### 2.2.1 热用户按需供热

**功能描述**: 基于室内温度预测和用户行为分析实现按需供热，避免过度供热或供热不足。

**页面/弹出页**:
| 页面/弹窗 | 功能说明 |
|-----------|----------|
| 用户列表页面 | 分页展示、筛选、搜索热用户 |
| 室温分布图 | 实时温度分布热力图 |
| 温度预测曲线弹窗 | 展示LSTM预测的未来温度曲线 |
| 供热策略配置弹窗 | 配置按需/定时/恒温供热策略 |
| 用户详情弹窗 | 查看历史温度曲线、预测结果 |

**核心功能**:
- 室内温度实时监测
- 温度预测（支持2小时超短期预测）
- 用户行为分析
- 供热需求计算
- 策略切换（按需/定时/恒温）

**算法设计**:

```
算法名称: LSTM温度预测算法
算法用途: 预测用户室内温度变化趋势
输入参数:
  - 历史室温数据序列 T = [t1, t2, ..., tn]
  - 室外温度序列 O = [o1, o2, ..., on]
  - 时间特征（小时、星期）
  - 建筑热特性参数

算法步骤:
1. 数据预处理
   1.1 缺失值填补（线性插值）
   1.2 归一化处理（Min-Max）
   1.3 特征工程（添加时间特征、滞后特征）

2. LSTM模型构建
   2.1 输入层: 序列长度=24, 特征维度=4
   2.2 LSTM层: 隐藏单元数=64, dropout=0.2
   2.3 全连接层: 输出维度=预测步长

3. 模型训练
   3.1 损失函数: MSE
   3.2 优化器: Adam (lr=0.001)
   3.3 早停策略: patience=5

4. 预测输出
   4.1 输出未来2-24小时温度预测值
   4.2 置信区间估计

数据要求:
- 训练数据: 至少30天的分钟级温度数据
- 采样频率: 建议15分钟
- 数据质量: 缺失率<10%
```

**功能描述**: 通过智能优化算法实现二次管网水力平衡，解决远近端冷热不均问题。

**页面/弹出页**:
| 页面/弹窗 | 功能说明 |
|-----------|----------|
| 二网平衡监控页面 | 管网拓扑、阀门状态实时展示 |
| 优化结果展示页面 | 遗传算法/粒子群优化结果 |
| 阀门调节配置弹窗 | 配置阀门调节参数 |
| 优化参数设置弹窗 | 设置种群规模、迭代次数等 |
| 优化历史记录弹窗 | 查看历史优化记录 |

**核心功能**:
- 管网水力计算
- 平衡度评估
- 优化计算（遗传算法/粒子群）
- 阀门开度建议
- 调节指令下发

**算法设计**:

```
算法名称: 遗传算法-二网平衡优化
算法用途: 计算管网最优阀门开度组合
输入参数:
  - 管网拓扑结构（节点、管道）
  - 各支路设计流量 Qi
  - 各阀门可调范围 [min_i, max_i]
  - 目标函数权重

算法步骤:
1. 初始化种群
   1.1 编码: 实数编码，每个基因代表一个阀门开度
   1.2 种群规模: 100
   1.3 初始解: 随机生成满足约束的阀门开度

2. 适应度计算
   2.1 水力计算: 求解管网方程得到实际流量
   2.2 目标函数: 
      min F = Σ|Qi_actual - Qi_design| / Qi_design
   2.3 惩罚项: 违反约束时增加惩罚值

3. 遗传操作
   3.1 选择: 锦标赛选择（size=5）
   3.2 交叉: 算术交叉（pc=0.8）
   3.3 变异: 高斯变异（pm=0.1）

4. 终止条件
   4.1 最大迭代次数: 500
   4.2 收敛判断: 连续50代最优解无改善

数据要求:
- 管网拓扑数据: 节点坐标、管段长度/管径/阻力系数
- 实时数据: 温度、压力、流量（5分钟更新）
- 阀门参数: 开度范围、调节精度
```

- PID自动调节
- 控制模式切换（自动/手动）
- 运行参数设定
- 历史数据查询

**算法设计**:

```
算法名称: 模糊PID控制算法
算法用途: 换热站温度自适应控制
输入参数:
  - 当前温度 T_current
  - 设定温度 T_setpoint
  - 历史误差序列 E = [e1, e2, ..., en]

算法步骤:
1. 模糊化处理
   1.1 计算误差 e = T_setpoint - T_current
   1.2 计算误差变化率 ec = de/dt
   1.3 将 e 和 ec 模糊化为语言变量

2. 模糊推理
   2.1 构建模糊规则库（49条规则）
   2.2 使用Mamdani推理方法
   2.3 输出PID参数增量 ΔKp, ΔKi, ΔKd

3. 参数整定
   3.1 Kp = Kp0 + ΔKp
   3.2 Ki = Ki0 + ΔKi
   3.3 Kd = Kd0 + ΔKd

4. 控制输出
   3.1 u(k) = u(k-1) + Kp*ec + Ki*e + Kd*(e-2e(k-1)+e(k-2))

数据要求:
- 传感器数据: 温度精度±0.1°C, 压力±0.01MPa
- 控制周期: 1-5分钟
- 历史数据: 最近30天
```

**算法设计**:

```
算法名称: 气候补偿曲线计算
算法用途: 确定不同室外温度下的供水温度设定值
输入参数:
  - 典型建筑热负荷特性
  - 室内设定温度 T_indoor
  - 室外温度范围 T_outdoor = [-20°C, 15°C]

算法步骤:
1. 热负荷计算
   1.1 公式: Q = K * F * (T_indoor - T_outdoor)
   1.2 K: 传热系数, F: 供热面积

2. 供水温度计算
   2.1 逆推法: 根据热负荷计算所需供水温度
   2.2 考虑系统特性修正

3. 曲线拟合
   3.1 采集典型工况点
   3.2 最小二乘法拟合
   3.3 线性/指数模型选择

数据要求:
- 建筑热工参数: 传热系数、供热面积
- 历史运行数据: 至少一个采暖季
- 室外温度: 小时级数据
```

**算法设计**:

```
算法名称: 能效指标计算
算法用途: 计算换热站能效指标
输入参数:
  - 一次侧: 供水温度T1s, 回水温度T1r, 流量F1
  - 二次侧: 供水温度T2s, 回水温度T2r, 流量F2

算法步骤:
1. 热量计算
   1.1 热量输入: Q_in = F1 × Cp × (T1s - T1r)
   1.2 热量输出: Q_out = F2 × Cp × (T2s - T2r)
   1.3 单位: MJ 或 GJ

2. 能效比计算
   2.1 COP = Q_out / Q_in
   2.2 标准: COP > 2.5 为合格

3. 热损失率
   3.1 Loss = (Q_in - Q_out) / Q_in × 100%
   3.2 标准: Loss < 15% 为合格

4. 供热均衡度
   4.1 计算各用户端温度标准差
   4.2 Balance = (1 - σ/mean) × 100%

数据要求:
- 温度数据: 精度±0.1°C
- 流量数据: 精度±2%
- 计算周期: 小时/日
```

#### 2.5.2 水力仿真

**功能描述**: 基于图论和牛顿迭代法进行管网水力计算。

**页面/弹出页**:
| 页面/弹窗 | 功能说明 |
|-----------|----------|
| 仿真配置页面 | 配置仿真参数 |

| 管网拓扑编辑弹窗 | 编辑管网拓扑 |

**核心功能**:
- 管网拓扑建模
- 水力仿真计算
- 工况模拟
- 结果可视化

**算法设计**:

```
算法名称: Newton-Raphson管网水力计算
算法用途: 求解管网节点压力和流量分布
输入参数:
  - 管网拓扑: 节点数N, 管段数M
  - 管段参数: 阻力系数S
  - 边界条件: 已知压力/流量节点

算法步骤:
1. 建立方程组
   1.1 节点连续方程: ΣQ_i = 0
   1.2 管段压降方程: h = S × Q²

2. Newton-Raphson迭代
   2.1 初始猜设: 所有节点压力
   2.2 残差计算: F(x) = A × Q - b
   2.3 雅可比矩阵: J = dF/dx
   2.4 迭代: x_new = x - J⁻¹ × F

3. 收敛判断
   3.1 最大残差 < ε1
   3.2 迭代次数 < max_iter

数据要求:
- 管网拓扑: 完整节点和管段数据
- 管段参数: 实测或设计阻力系数
- 计算精度: 压力±0.5kPa
```

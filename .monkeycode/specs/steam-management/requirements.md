# 蒸汽管理系统需求文档

需求名称：steam-management
更新日期：2026-03-16

## 1. 业务背景

蒸汽管理系统面向工业蒸汽锅炉用户，通过智能化手段实现蒸汽系统的节能降耗和高效运行。系统需要采集用户生产数据、分析用汽规律、优化锅炉运行策略，并提供全面的能耗监控管理功能。

## 2. 功能需求

### 2.1 数据采集与存储

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| REQ-001 | 采集蒸汽锅炉运行数据（压力、温度、流量） | P0 |
| REQ-002 | 采集用水量、用电量数据 | P0 |
| REQ-003 | 采集设备状态数据（启停、故障） | P1 |
| REQ-004 | 数据存储到时序数据库，支持历史查询 | P0 |
| REQ-005 | 实时数据缓存到Redis | P1 |

### 2.2 用户行为画像

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| REQ-006 | 基于生产数据生成用户行为画像 | P0 |
| REQ-007 | 分析生产规律（工作时段、峰值时段） | P0 |
| REQ-008 | 统计水电耗等能耗数据 | P0 |
| REQ-009 | 能耗对标分析（同类企业对比） | P1 |
| REQ-010 | 画像定期更新（每日/每周） | P1 |

### 2.3 锅炉运行策略

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| REQ-011 | 根据画像生成运行功率策略 | P0 |
| REQ-012 | 生成启停调度计划 | P0 |
| REQ-013 | 多锅炉协同运行优化 | P1 |
| REQ-014 | 策略效果评估与反馈 | P1 |
| REQ-015 | 策略手动调整与下发 | P1 |

### 2.4 能耗监控管理

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| REQ-016 | 实时能耗监控大屏 | P0 |
| REQ-017 | 日/周/月/年能耗报表 | P0 |
| REQ-018 | 能耗异常告警 | P0 |
| REQ-019 | 设备故障预警 | P1 |
| REQ-020 | 能耗成本分析 | P1 |

### 3.2 画像分析接口

| API-006 | GET | /api/steam/profile/{enterpriseId}/energy-consumption | 获取能耗统计 |

## 4. 性能需求

| 指标 | 目标值 |
|------|--------|
| 数据采集成功率 | ≥99.5% |
| 实时数据延迟 | ≤10秒 |
| 策略计算时间 | ≤30秒 |
| 系统响应时间 | ≤2秒 |
| 并发支持 | ≥100设备 |

## 5. 验收标准

1. 设备数据能够正确采集并存储，查询历史数据功能正常
2. 用户画像能够正确生成，包含生产规律和能耗特征
3. 运行策略能够根据画像生成优化建议
4. 能耗监控大屏实时展示数据，报表能够正确生成
5. 告警功能能够正确触发并通知
6. 系统性能满足要求

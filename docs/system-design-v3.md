## 1 系统概述

### 1.1 项目背景与目标

锅炉集中供热智慧管理系统是一套面向城市集中供热企业的综合性智能管理平台。系统采用物联网、大数据、人工智能等先进技术，实现对供热全过程的智能化监控、精细化管理和优化调控。系统覆盖从热源生产、一次管网输送、换热站转换到二次管网分配直至终端用户的完整供热链条，为供热企业提供数字化、智能化、网络化的管理手段。

本系统的核心目标包括：一是提高供热系统的运行效率，通过负荷预测和智能调度实现能源的合理配置；二是提升供热服务质量，通过室温监测和按需供热满足用户的个性化需求；三是降低运营成本，通过管网平衡、气候补偿等技术手段减少能源浪费；四是预防故障发生，通过异常检测和风险评估实现故障的提前预警。

### 1.2 系统架构设计

系统采用前后端分离的微服务架构设计。前端基于Vue3框架构建响应式管理界面，使用Vite作为构建工具，Pinia进行状态管理，ECharts实现数据可视化，Element Plus作为UI组件库。后端采用Spring Cloud微服务架构，支持高并发、高可用的分布式部署。

系统功能模块按照业务领域划分为八大类：监控中心、热用户管理、换热站管理、管网控制、热源管理、智能优化、分析预测、系统管理。每个大类下包含若干具体功能页面，共24个功能页面，覆盖了供热系统管理的各个环节。

#### 2.2.2 核心算法：用户室温预测算法

用户室温预测算法基于时间序列分析和机器学习方法实现。算法首先采集用户室温和环境参数的历史数据，然后构建预测模型，最后输出未来一定时间内的温度预测值。

**算法步骤：**

步骤一：数据采集与预处理。采集过去24小时的室温和环境数据，包括室外温度、供热时间、建筑物特性等。对采集的数据进行缺失值填补、异常值剔除、归一化处理。

步骤二：特征工程。将时间序列数据转换为监督学习数据集，采用滑动窗口方法构建样本。输入特征包括：历史室温序列、室外温度、时间特征（小时、星期）、建筑热特性参数。输出为未来N个时间点的室温预测值。

步骤三：模型训练。采用LSTM（长短期记忆网络）作为预测模型。LSTM是一种特殊的循环神经网络，能够学习时间序列中的长期依赖关系。网络结构包含输入层、LSTM隐藏层、全连接输出层。

步骤四：模型预测。将当前时刻的数据输入训练好的模型，输出未来2-6小时的温度预测值。同时通过多次预测的方差计算置信区间。

**算法公式：**

LSTM单元的前向传播计算公式：

遗忘门：$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$

输入门：$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$

候选细胞：$\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$

细胞更新：$C_t = f_t \cdot C_{t-1} + i_t \cdot \tilde{C}_t$

输出门：$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$

隐藏状态：$h_t = o_t \cdot \tanh(C_t)$

其中，$\sigma$为Sigmoid激活函数，$\tanh$为双曲正切激活函数，$W$和$b$为网络权重和偏置。

#### 2.3.2 核心算法：遗传算法

遗传算法是一种模拟自然选择和遗传机制的优化算法，用于求解复杂的非线性优化问题。在二网平衡优化中，遗传算法用于寻找最优的阀门开度组合。

**算法步骤：**

步骤一：编码。将每个阀门的开度（0-100%）编码为染色体中的一个基因，整条染色体表示一个完整的阀门开度方案。

步骤二：初始化。随机生成初始种群，种群中的每个个体代表一种阀门开度组合方案。种群规模通常设置为50-100。

步骤三：适应度计算。对于每个个体，计算其适应度值。适应度函数定义为室内温度偏差的倒数，偏差越小适应度越高。约束条件包括：阀门开度范围、流量平衡方程、管网压力约束。

步骤四：选择。根据适应度值采用轮盘赌算法选择父代个体，适应度高的个体被选中的概率更大。

步骤五：交叉。以一定概率（通常0.7-0.9）对父代个体进行单点交叉，产生子代个体。

步骤六：变异。以一定概率（通常0.01-0.05）对子代个体的基因进行随机变异，增加种群的多样性。

步骤七：精英保留。将父代中适应度最高的个体直接保留到子代，确保最优解不会丢失。

步骤八：终止判断。达到最大迭代次数或适应度值不再显著改善时停止，否则返回步骤三。

**算法公式：**

轮盘赌选择概率：$P_i = \frac{f_i}{\sum_{j=1}^{N} f_j}$

其中，$f_i$为第$i$个个体的适应度值，$N$为种群规模。

交叉操作：$child_1[k] = parent_1[k], child_2[k] = parent_2[k]$（$k < crossover\_point$）

变异操作：$gene' = gene + \Delta$，其中$\Delta$为随机扰动

#### 2.5.2 核心算法：PID控制算法

PID（比例-积分-微分）控制是过程控制中应用最广泛的控制算法，通过对偏差进行比例、积分、微分运算得到控制输出，实现对被控对象的精确调节。

**算法步骤：**

步骤一：偏差计算。计算当前测量值与设定值之间的偏差$e(t) = setpoint - measured\_value$。

步骤二：比例运算。将偏差乘以比例系数$K_p$得到比例项输出：$P_{out} = K_p \cdot e(t)$。比例项根据偏差大小进行调节，偏差越大，调节作用越强。

步骤三：积分运算。对偏差进行积分运算：$I_{out} = K_i \cdot \int_0^t e(\tau) d\tau$。积分项消除静差，提高控制精度。

步骤四：微分运算。计算偏差的变化率：$D_{out} = K_d \cdot \frac{de(t)}{dt}$。微分项预测偏差变化趋势，提前进行调节，减少超调。

步骤五：控制输出。将三项输出叠加得到最终控制输出：$u(t) = P_{out} + I_{out} + D_{out}$。

步骤六：输出限幅。将控制输出限制在合理范围内（如0-100%），防止执行机构超限。

**算法公式：**

连续PID：$u(t) = K_p e(t) + K_i \int_0^t e(\tau) d\tau + K_d \frac{de(t)}{dt}$

离散PID：$u(k) = K_p e(k) + K_i \sum_{i=0}^{k} e(i) \cdot \Delta t + K_d \frac{e(k) - e(k-1)}{\Delta t}$

#### 2.6.2 核心算法：LSTM负荷预测算法

负荷预测采用LSTM深度学习算法，结合气象数据和历史负荷数据，实现高准确度的负荷预测。

**算法步骤：**

步骤一：数据收集。收集过去30天以上的历史负荷数据、同期的气象数据（室外温度、湿度、风速）、日历特征数据（星期、是否节假日）。

步骤二：数据预处理。对原始数据进行清洗、归一化处理。将数据划分为训练集和测试集，通常按时间顺序前80%作为训练集，后20%作为测试集。

步骤三：特征构建。将时间序列数据转换为监督学习格式。输入特征包括：历史负荷序列（过去N个时间点）、室外温度、时间特征（小时、星期）、气象特征。输出为未来M个时间点的负荷预测值。

步骤四：模型构建。构建LSTM网络模型，包含输入层、LSTM隐藏层（多层）、Dropout层、全连接输出层。

步骤五：模型训练。使用训练数据对模型进行训练，通过反向传播算法优化网络参数。训练过程中使用验证集监控模型性能，防止过拟合。

步骤六：模型预测。使用训练好的模型对未来负荷进行预测。输出预测值的同时，通过模型的不确定性估计计算置信区间。

**输入特征：**

| 特征名称 | 数据类型 | 说明 |
|----------|----------|------|
| historical_load | Float[] | 历史负荷序列（过去24个时间点） |
| outdoor_temp | Float | 室外温度 |
| humidity | Float | 相对湿度 |
| wind_speed | Float | 风速 |
| hour | Integer | 小时（0-23） |
| day_of_week | Integer | 星期（0-6） |
| is_holiday | Boolean | 是否节假日 |

**算法公式：**

LSTM单元计算公式见2.2.2节。

预测准确率评估采用MAPE（平均绝对百分比误差）：

$$MAPE = \frac{1}{n} \sum_{i=1}^{n} \left| \frac{y_i - \hat{y}_i}{y_i} \right| \times 100\%$$

#### 2.7.2 核心算法：孤立森林异常检测

孤立森林（Isolation Forest）是一种基于决策树的异常检测算法，其核心思想是异常点在特征空间中更容易被"孤立"。

**算法步骤：**

步骤一：构建孤立树。从样本数据中随机选择一个特征和该特征的随机切分值，将样本空间递归划分为两个子空间。递归终止条件：树的深度达到限定值或子空间样本数少于最小值。

步骤二：构建孤立森林。独立构建多棵孤立树，形成一片森林。每棵树的划分过程相互独立，确保随机性。

步骤三：计算异常分数。对于新样本$x$，计算其在各棵树中的平均路径长度$E(h(x))$，然后计算异常分数：

$$s(x, n) = 2^{-\frac{E(h(x))}{c(n)}}$$

其中，$c(n)$为样本数为$n$时的平均路径长度归一化因子。

步骤四：判定异常。设定阈值$\theta$（通常为0.6-0.8），当$s(x, n) > \theta$时，判定样本为异常。

**算法公式：**

平均路径长度：$c(n) = 2H(n-1) - \frac{2(n-1)}{n}$

其中，$H(n-1)$为调和数，可用$\ln(n-1) + 0.5772156649$近似。

异常分数：$s(x, n) = 2^{-\frac{E(h(x))}{c(n)}}$

当$s(x, n)$接近1时，样本为异常的概率高；当$s(x, n)$接近0时，样本为正常的概率高。

**热用户管理**（/users）：热用户基本信息的CRUD操作界面，包含用户编码、姓名、联系方式、所属建筑物、供热面积等字段的增删改查。

**建筑物管理**（/buildings）：建筑物信息管理界面，包含建筑物名称、地址、建筑面积、供热面积、设计参数等信息。

**换热站管理**（/stations）：换热站基本信息管理界面，包含站点编码、名称、位置、设计能力、投运日期等。

**告警管理**（/alarms）：系统告警信息的集中管理界面，支持告警的查看、确认、处理、关闭等操作流程。

**供能设备管理**（/equipment）：锅炉、循环泵等主要设备的信息管理，包含设备台账、维护记录等信息。

**趋势分析**（/trend-analysis）：历史数据的趋势分析功能，展示温度、压力、流量、能耗等指标的变化趋势。

**风险评估**（/risk-assessment）：管网和设备的风险量化评估，识别潜在风险并生成风险等级报告。

**分时分区控制**（/time-zone-control）：根据不同区域、不同时间段的供热需求差异制定差异化控制策略。

**一网控制策略**（/primary-network）：一次管网的控制策略配置和监控界面。

**水力仿真**（/hydraulic-simulation）：管网水力仿真计算工具，支持静态仿真和动态仿真两种模式。

**智能电动调节**（/valve-control）：电动调节阀的远程控制和状态监测界面。

## 5 系统接口

### 5.1 前端组件接口规范

前端组件采用Vue3的Composition API风格，通过props接收数据，通过emit发送事件。主要组件类型包括：

**数据展示类组件**：接收数据源、显示格式配置、筛选条件等props，输出展示后的视图。
**控制类组件**：接收当前状态、控制参数、权限标识等props，输出用户的操作行为事件。
**图表类组件**：接收图表类型配置、数据源、样式配置等props，输出交互事件和导出操作。

### 5.2 API接口规范

后端API采用RESTful风格设计，遵循统一的接口规范：

**接口路径**：以资源名称命名，如`/api/users`、`/api/stations`、`/api/alarms`

**HTTP方法**：
- GET：查询资源
- POST：创建新资源
- PUT：更新资源
- DELETE：删除资源

**响应格式**：
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**认证方式**：采用Token认证，用户登录成功后获取Token，后续请求在请求头中携带Token。

## 6 安全设计

### 6.1 权限控制

系统采用基于角色的访问控制（RBAC）模型：

**角色定义**：
- 管理员：系统全部权限
- 调度员：监控、调度相关权限
- 运维员：设备、告警相关权限
- 访客：只读权限

### 6.2 操作审计

系统对所有敏感操作进行审计记录，包括：
- 用户登录/登出
- 数据增删改
- 系统参数修改
- 控制指令下发

审计日志包含：操作时间、操作人员、操作类型、操作内容、操作结果、来源IP。

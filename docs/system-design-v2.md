## 1 系统概述

### 1.1 项目背景

随着城市化进程的加速推进，集中供热作为北方地区重要的民生工程，其运行效率和智能化水平直接关系到居民生活质量和企业经济效益。传统的供热系统普遍存在管理粗放、能耗偏高、调控滞后等问题，难以满足现代智慧城市建设的需求。锅炉集中供热智慧管理系统旨在通过物联网、大数据、人工智能等先进技术，实现供热全过程的智能化监控、精细化管理和优化调控，系统覆盖从热源生产、一次管网输送、换热站转换到二次管网分配直至终端用户的完整供热链条。

本系统采用前后端分离的微服务架构设计，前端基于Vue3框架构建响应式管理界面，后端采用Spring Boot微服务架构。系统核心功能包括实时数据采集与监控、负荷预测与动态调度、管网水力仿真、二网平衡控制、异常检测与预警、多能互补优化等模块。通过机器学习算法对历史数据进行分析建模，系统能够实现提前预测供热需求、自动调整运行参数、主动发现潜在故障，从而显著提升供热系统的运行效率和服务质量。

### 1.2 系统架构

本系统采用分层架构设计，从下至上依次分为数据采集层、数据处理层、业务应用层和展示交互层。数据采集层负责从各类传感器、智能仪表和远程终端单元获取实时数据；数据处理层完成数据清洗、存储、计算和分析；业务应用层实现供热生产调度、管网平衡控制、负荷预测优化等核心业务逻辑；展示交互层提供可视化的监控大屏和管理操作界面。各层之间通过标准化的API接口进行数据交互，确保系统的可扩展性和维护性。

```
┌─────────────────────────────────────────────────────────────┐
│                      展示交互层                              │
│  (监控大屏 / 用户管理 / 换热站管理 / 智能优化 / 分析预测)      │
├─────────────────────────────────────────────────────────────┤
│                      业务应用层                              │
│  (热源调控 / 管网控制 / 负荷预测 / 异常检测 / 能耗优化)        │
├─────────────────────────────────────────────────────────────┤
│                      数据处理层                              │
│  (数据清洗 / 实时计算 / 机器学习 / 模型推理 / 时序分析)        │
├─────────────────────────────────────────────────────────────┤
│                      数据采集层                              │
│  (温度传感器 / 压力变送器 / 流量计 / 智能仪表 / RTU)          │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 技术选型

前端技术栈选用Vue3作为核心框架，结合Vite构建工具实现快速开发和热模块替换。状态管理采用Pinia框架，路由管理使用Vue Router，网络请求使用Axios库。图表可视化选用ECharts组件库，UI组件库采用Element Plus确保界面的一致性和易用性。后端技术栈基于Spring Cloud微服务架构，使用Redis进行缓存加速、MySQL进行持久化存储，并引入机器学习引擎支持负荷预测和异常检测等智能分析功能。

## 2 一级模块划分

根据供热系统的业务特点和管理需求，系统功能划分为八大一级模块，分别对应不同的业务领域和管理职能。这种模块划分既考虑了功能的内聚性，也兼顾了业务流程的连贯性，确保各模块既能独立运行又能协同工作。每个一级模块下包含若干二级模块，形成层次分明的功能结构，便于用户快速定位所需功能，也便于开发团队进行任务分配和版本迭代。

| 2 | 热用户管理 | M2 | 热用户信息管理、按需供热控制 |
| 3 | 换热站管理 | M3 | 换热站设备管理、自控系统监控 |

## 3 二级模块详细说明

### 3.1 监控中心模块

监控中心作为系统的门户和首脑，负责呈现整个供热系统的运行全景。该模块以数据大屏的形式展示关键运行指标，包括换热站数量、建筑物数量、热用户数量、告警数量等统计信息，以及温度趋势、负荷曲线、运行状态分布等可视化图表。监控中心的数据来源于各个子系统的实时上报，经过数据处理层的聚合计算后呈现，确保用户能够在最短时间内掌握系统整体运行状态。

监控中心的设计遵循重要性优先原则，将最关键的指标放置在显眼位置，次要信息通过折叠或切换方式呈现。温度趋势监控图表展示一次侧供回水温度和二次侧供回水温度的24小时变化曲线，帮助调度人员判断系统运行是否正常；换热站运行状态饼图直观展示各换热站的运行、停止、故障状态分布；热负荷曲线图呈现系统的实时负荷和历史对比，为调度决策提供数据支撑；最近告警列表实时滚动显示系统产生的各类告警信息，便于快速响应处理。

### 3.2 热用户管理模块

热用户管理模块聚焦于终端热用户的供热服务质量和精细化调控需求。该模块包含三个二级模块：热用户管理、建筑物管理、热用户按需供热。热用户管理实现热用户基本信息的增删改查，包括用户编码、姓名、联系方式、所属建筑物、供热面积等字段；建筑物管理维护建筑物的结构信息、供热面积、设计参数等；热用户按需供热是核心功能模块，基于用户室内温度采集数据和目标温度设定，通过智能算法实现按需供热调控。

按需供热的核心思想是根据用户的实际供热需求动态调整供热策略，避免过度供热造成的能源浪费和用户不适。系统实时采集用户室内温度，与目标温度进行对比，结合房屋的保温性能、历史用热习惯等因素，计算最优的供热参数。对于住宅用户，系统支持远程温控阀控制，可根据用户在家状态和设定温度自动调节；对于商业用户，系统提供分时段供热策略，在保障舒适度的前提下降低非工作时段的能耗。

### 3.3 换热站管理模块

换热站是连接一次管网和二次管网的关键枢纽，其运行状态直接决定着供热质量和系统效率。该模块包含换热站管理和换热站自控两个二级模块。换热站管理负责换热站基本信息的维护，包括站点编码、名称、位置、设计能力、投运日期等；换热站自控模块实时监控换热站的运行参数，包括一次侧供回水温度压力、二次侧供回水温度压力、循环泵运行状态、变频器频率等，并通过自控逻辑实现无人值守运行。

换热站自控系统采用先进的PID控制算法，根据二次侧供水温度设定值与实际值的偏差自动调节一次侧调节阀开度，实现稳定供水温度控制。同时，系统具备多种运行模式切换功能，包括定温度运行、定流量运行、室外温度补偿运行等，可根据外部条件和运行需求灵活切换。故障自诊断功能能够实时监测设备运行状态，发现异常及时告警并采取保护措施，确保设备安全和系统稳定。

### 3.4 管网控制模块

管网控制模块负责一次管网和二次管网的监控、调节和平衡控制，是实现全网均衡供热的核心模块。该模块包含六个二级模块：一网控制策略、二网平衡、管网平衡策略、智能电动调节、气候补偿模式、二级网控温。一次管网控制策略模块实现一次管网的流量分配和温度调节，确保各换热站获得所需的热量；二网平衡模块是实现均匀供热的关键，通过调节各支路的阀门开度使各用户室温保持在设定范围内。

管网平衡策略模块提供多种平衡调节算法，包括比例调节法、积分调节法、模糊控制法等，用户可根据实际情况选择合适的策略。智能电动调节模块实现对电动调节阀的远程控制和状态监测，支持就地/远程模式切换、阀位反馈、故障诊断等功能。气候补偿模块根据室外温度变化自动调整供热曲线，实现根据天气条件动态供热的节能运行。二级网控温模块专门针对二次管网的温度进行精细化控制，通过压差控制、流量控制等手段确保供热效果。

### 3.5 热源管理模块

热源管理模块负责锅炉房和热源侧的生产调度管理，是整个供热系统的动力源头。该模块包含热源调控和供能设备管理两个二级模块。热源调控模块监控各锅炉的运行状态和运行参数，包括负荷率、供水温度、回水温度、运行效率等，并通过智能调度算法优化多台锅炉的运行组合，在满足供热需求的前提下最大化能源利用效率。供能设备管理模块维护锅炉、循环泵、补水泵等主要设备的基本信息和维护记录。

热源调控模块集成了LSTM神经网络预测模型，基于室外温度、时间因素、历史负荷等多维输入特征，预测未来4小时的热负荷需求，为锅炉调度提供科学依据。系统支持锅炉的启停控制、负荷调节、运行模式切换等操作，具备自动和手动两种控制模式。在自动模式下，系统根据预测负荷自动决定各锅炉的运行状态和出力，实现经济运行；在手动模式下，调度人员可人工干预调整，确保特殊情况下的人工可控。

### 3.6 智能优化模块

智能优化模块运用人工智能和优化算法实现供热系统的运行优化，是系统的智慧大脑。该模块包含四个二级模块：负荷预测、多能耦合寻优、分时分区控制、管网平衡分析。负荷预测模块运用机器学习算法对短期、中期的热负荷进行预测，为调度计划提供数据支撑；多能耦合寻优模块综合考虑电、热、冷等多种能源的供给和价格，优化能源利用结构；分时分区控制模块根据不同区域、不同时间段的供热需求差异制定差异化调控策略。

负荷预测模块支持超短期（2小时）、短期（24小时）、中期（7天）三种预测周期，采用时间序列分析和神经网络算法相结合的方式实现高准确度预测。预测结果以曲线图形式展示，包含实际负荷、预测负荷和置信区间，帮助调度人员了解预测的可信度。多能耦合寻优模块考虑燃气锅炉、电锅炉、空气源热泵等多种供热设备的特性和成本，构建优化模型求解最优设备组合和运行策略。分时分区控制模块结合用户的作息规律和建筑物的热惰性，实现蓄热式供热和错峰运行。

### 3.7 分析预测模块

分析预测模块通过对历史数据的深度分析发现规律、预测趋势、识别异常，为运行优化和故障预防提供决策支持。该模块包含四个二级模块：趋势分析、异常状态预测、风险评估、气象分析。趋势分析模块对温度、压力、流量、能耗等关键指标进行历史趋势分析，发现周期性规律和异常波动；异常状态预测模块基于机器学习算法识别设备故障和运行异常的早期信号；风险评估模块对管网泄漏、设备故障等风险进行量化评估；气象分析模块研究与供热相关的天气因素变化规律。

异常状态预测模块采用基于时序数据的异常检测算法，能够识别温度突变、压力异常、流量偏离等不正常状态，提前数小时预测可能发生的故障，为预防性维护提供时间窗口。风险评估模块综合考虑设备役龄、运行工况、历史故障记录等因素，计算各设备的风险等级，帮助维护人员优先关注高风险设备。气象分析模块接入气象数据接口，获取温度、湿度、风力、天气状况等预报信息，为负荷预测和调度优化提供输入。

### 3.8 系统管理模块

系统管理模块提供系统运行所需的基础数据维护和运维支撑功能，确保系统持续稳定运行。该模块包含告警管理和设备管理两个二级模块。告警管理模块接收来自各子系统的告警信息，按照告警级别进行分类和优先级排序，支持告警确认、处理、关闭等流程，并提供告警统计分析功能帮助识别频繁发生的告警类型。设备管理模块维护系统中各类设备的基本信息台账，包括设备编码、名称、型号、厂商、安装位置、投运日期等，并记录设备的维修保养历史。

告警管理采用分级处理机制，将告警分为信息、警告、严重三个级别，不同级别的告警采用不同的展示方式和处理要求。系统支持多种告警通知方式，包括界面弹窗、声音提示、短信通知等，确保重要告警能够及时送达相关人员。设备管理模块与备件库存管理联动，支持设备维修工单的创建和跟踪，实现设备全生命周期的信息化管理。

## 4 功能页面详细说明

### 4.1 监控大屏页面

监控大屏是系统登录后的默认首页，以数据可视化的方式呈现供热系统的整体运行状态。页面采用卡片式布局，顶部的四个统计卡片分别显示换热站数量、建筑物数量、热用户数量、告警数量四个核心指标，点击可跳转到相应的管理模块查看详情。中部左侧为温度趋势监控图，以折线图形式展示一次侧供回水和二次侧供回水的24小时温度变化曲线，不同曲线采用不同颜色区分，鼠标悬停可显示具体数值。

页面中部右侧为换热站运行状态饼图，直观展示运行中、停止、故障三种状态的换热站数量分布，不同状态采用绿色、灰色、红色区分。下部左侧为热负荷曲线柱状图，展示各时段的系统热负荷；下部右侧为最近告警表格，列出最新产生的告警记录，包含告警类型、告警信息、告警级别、发生时间等字段，点击告警可查看详细信息并进行处理。页面支持数据自动刷新，默认刷新间隔为30秒，也可手动点击刷新按钮获取最新数据。

### 4.2 热用户按需供热页面

热用户按需供热页面是实现精细化供热管理的核心操作界面，页面分为四个功能区域。左侧区域为用户温度分布饼图，展示温度正常、偏低、偏高的用户比例分布，不同温度状态采用绿色、蓝色、红色区分。右侧区域为实时温度监控表格，列出所有热用户的当前温度、目标温度、采集时间等数据，表格支持按建筑物筛选，温度数值根据与目标温度的偏差自动着色显示，超出允许范围的显示为红色。

页面中部为温度趋势预测图，展示选中用户的温度历史曲线和未来预测曲线，预测时间可选择2小时、4小时、6小时三档，预测结果包含预测值和置信区间，帮助判断未来温度变化趋势。下部区域为供热需求统计柱状图，按建筑物展示热负荷需求分布，为调度计划提供参考。页面提供温度预测按钮，点击可选中特定用户查看详细的温度预测数据，以对话框形式呈现包含预测时间、预测温度、置信区间、置信度的数据列表。

### 4.3 热源调控页面

热源调控页面展示锅炉房各锅炉的运行状态并进行调度控制，页面布局分为三个区域。顶部为锅炉卡片区，以卡片形式展示各台锅炉的基本信息，包括锅炉名称、运行状态、负荷率、供水温度、回水温度、运行效率等，每台锅炉配有增加负荷和减少负荷两个控制按钮，点击可调整锅炉出力。页面支持显示多台锅炉，卡片根据运行状态显示不同的边框颜色。

## 5 核心算法详细说明

### 5.1 负荷预测算法

负荷预测是供热系统智能化运行的核心基础，准确预测未来负荷对于优化调度、节能降耗具有重要意义。本系统采用基于LSTM（长短期记忆网络）与传统时间序列分析相结合的混合预测方法，充分利用历史数据的时序特性和多源数据的环境因素，实现高准确度的负荷预测。

#### 5.1.1 算法原理

LSTM是一种特殊的循环神经网络结构，通过引入门控机制解决了传统RNN在处理长序列时存在的梯度消失和梯度爆炸问题。LSTM网络包含输入门、遗忘门、输出门三个门控单元和细胞状态，可以选择性记住长期依赖信息，非常适合处理时间序列预测问题。在负荷预测应用中，LSTM网络接收过去若干时间点的负荷序列作为输入，学习负荷变化的时序规律，然后输出未来时间点的负荷预测值。

预测模型的输入特征包括室外温度、历史负荷、时间特征（小时、星期、是否节假日）、日历特征（供暖季阶段）等。室外温度是影响供热负荷的最主要因素，LSTM网络能够学习室外温度与负荷之间的非线性映射关系。时间特征用于捕捉负荷的周期性变化规律，如工作日和休息日的负荷差异、夜间和白天的负荷差异等。模型输出为未来4小时的负荷预测值，同时输出预测的置信区间用于评估预测的可信度。

#### 5.1.2 算法流程

负荷预测算法的完整流程包括数据预处理、特征工程、模型训练、预测推理四个阶段。数据预处理阶段对原始数据进行缺失值填补、异常值剔除、噪声过滤等操作，确保数据质量。特征工程阶段将原始数据转换为模型可接受的输入格式，包括时间特征编码、归一化处理、滑动窗口构建等。模型训练阶段使用历史数据对LSTM网络进行训练，通过反向传播算法优化网络参数，训练完成后保存模型用于在线预测。

预测推理阶段的流程如下：首先获取当前时刻的实时负荷数据和天气预报的室外温度数据；然后构建输入样本，将最近若干时间点的负荷序列和对应的环境因素组成输入向量；接着将输入向量传入训练好的LSTM模型进行前向计算，得到预测输出；最后对输出进行反归一化处理，得到实际的负荷预测值，同时根据模型的预测误差分布计算置信区间。整个预测过程在后台自动执行，预测结果定期更新到前端界面供调度人员参考。

#### 5.1.3 算法公式

LSTM网络的前向传播计算涉及以下核心公式：

遗忘门计算公式：$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$

输入门计算公式：$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$

候选细胞状态公式：$\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$

细胞状态更新公式：$C_t = f_t \cdot C_{t-1} + i_t \cdot \tilde{C}_t$

输出门计算公式：$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$

隐藏状态输出公式：$h_t = o_t \cdot \tanh(C_t)$

其中，$\sigma$表示Sigmoid激活函数，$\tanh$表示双曲正切激活函数，$W$和$b$分别表示权重矩阵和偏置向量，$f_t$、$i_t$、$o_t$分别表示遗忘门、输入门、输出门的输出，$C_t$表示细胞状态，$h_t$表示隐藏状态。

预测准确率采用MAPE（平均绝对百分比误差）进行评估，计算公式如下：

$$MAPE = \frac{1}{n} \sum_{i=1}^{n} \left| \frac{y_i - \hat{y}_i}{y_i} \right| \times 100\%$$

其中，$y_i$表示实际负荷值，$\hat{y}_i$表示预测负荷值，$n$表示样本数量。

### 5.2 水力仿真算法

水力仿真是供热系统设计、运行分析和故障诊断的重要工具，通过求解管网的水力平衡方程组，计算各管段的流量分配和节点的压力分布。本系统提供静态仿真和动态仿真两种模式，静态仿真用于分析稳态运行工况，动态仿真用于模拟负荷变化过程中的过渡过程。

#### 5.2.1 算法原理

管网水力计算基于质量守恒和能量守恒定律。对于稳态流动，流入任一节点的流量等于流出该节点的流量（质量守恒），任意两节点之间的压力损失等于两节点之间的势能差（能量守恒）。根据这两个基本定律，可以建立管网的节点方程组或回路方程组，求解得到各管段的流量和节点的压力。

本系统采用节点法进行水力计算，以各节点的压力为未知量建立方程组。节点$i$的流量平衡方程为：

$$\sum_{j \in S_i} Q_{ij} = 0$$

其中，$S_i$表示与节点$i$相连的所有管段的集合，$Q_{ij}$表示从节点$i$流向节点$j$的流量。

管段的流量与压力损失之间的关系由阻力特性方程描述，遵循平方阻力定律：

$$Q = K \sqrt{\Delta P}$$

或近似为：

$$\Delta P = S Q^2$$

其中，$K$为管段的阻力系数，$S$为管段的阻抗，$Q$为流量，$\Delta P$为压力损失。

将上述方程联立，得到关于节点压力的非线性方程组，采用迭代法求解。

#### 5.2.2 Newton-Raphson算法

Newton-Raphson法是求解非线性方程组最常用的迭代方法，具有二阶收敛速度。本系统将其作为默认的迭代算法，计算步骤如下：

设节点压力向量为$P = [P_1, P_2, ..., P_n]^T$，节点流量平衡方程的残差向量为$F(P) = [f_1(P), f_2(P), ..., f_n(P)]^T$。Newton-Raphson法的迭代公式为：

$$P^{k+1} = P^k - J^{-1}(P^k) \cdot F(P^k)$$

其中，$J(P^k)$为雅可比矩阵，其元素定义为：

$$J_{ij} = \frac{\partial f_i}{\partial P_j}$$

迭代的收敛判据为：$||F(P^k)||_2 < \varepsilon$ 或 $||P^{k+1} - P^k||_2 < \varepsilon$，其中$\varepsilon$为指定的收敛精度。

雅可比矩阵中元素的计算涉及管段流量对节点压力的偏导数。根据阻力方程 $Q_{ij} = K_{ij} \cdot \text{sgn}(P_i - P_j) \cdot \sqrt{|P_i - P_j|}$，可得：

$$\frac{\partial Q_{ij}}{\partial P_i} = \frac{K_{ij}}{2 \sqrt{|P_i - P_j|}}$$

$$\frac{\partial Q_{ij}}{\partial P_j} = -\frac{K_{ij}}{2 \sqrt{|P_i - P_j|}}$$

#### 5.2.3 梯度下降算法

梯度下降法是另一种可用于求解非线性方程组的迭代算法，其原理是通过沿目标函数梯度的负方向迭代，逐步逼近最优解。虽然收敛速度较Newton-Raphson法慢，但梯度下降法对初始值不敏感，鲁棒性较好，适合作为Newton-Raphson法的补充。

设目标函数为各节点流量残差的平方和：

$$F(P) = \frac{1}{2} \sum_{i=1}^{n} f_i^2(P)$$

梯度下降法的迭代公式为：

$$P^{k+1} = P^k - \alpha^k \nabla F(P^k)$$

其中，$\alpha^k$为第$k$步的步长，可通过线搜索方法确定；$\nabla F(P)$为梯度向量，其元素为：

$$\frac{\partial F}{\partial P_j} = \sum_{i=1}^{n} f_i(P) \cdot \frac{\partial f_i}{\partial P_j}$$

#### 5.2.4 动态仿真算法

动态仿真用于模拟管网在负荷变化、阀门调节等扰动下的过渡过程，需要求解包含时间导数的动态方程组。动态水力过程的基本方程为：

$$\frac{\partial Q}{\partial t} + \frac{\partial P}{\partial x} + \frac{Q}{A} = 0$$

这是一维非恒定流方程，采用有限差分法进行离散求解。将管网各管段划分为若干计算单元，在时间和空间上进行差分逼近，得到递推方程组：

$$Q_i^{n+1} = Q_i^n - \frac{\Delta t}{\Delta x} (P_{i+1}^n - P_i^n) - \frac{\Delta t}{A_i} Q_i^n$$

其中，$\Delta t$为时间步长，$\Delta x$为空间步长，$A$为管段的流通截面积。上标表示时间层，下标表示空间位置。

动态仿真的稳定性要求满足CFL条件：$\Delta t \leq \frac{\Delta x}{c + |v|}$，其中$c$为水击波速，$v$为流速。本系统默认时间步长为5分钟，可在1-60分钟范围内调整。

### 5.3 二网平衡控制算法

二次管网平衡控制是实现均匀供热的核心技术，其目标是使各用户的室内温度保持在设定范围内，避免出现远近端冷热不均的现象。本系统提供多种平衡控制算法，包括比例调节法、PI调节法、模糊控制法等，用户可根据实际情况选择合适的算法。

#### 5.3.1 比例调节算法

比例调节是最简单的平衡控制方法，根据各用户室温与目标温度的偏差，按比例调整阀门开度。设用户$i$的室温偏差为$e_i = T_i^{actual} - T_i^{target}$，则阀门开度调整量为：

$$\Delta V_i = K_p \cdot e_i$$

其中，$K_p$为比例增益系数。调整后的阀门开度为：

$$V_i^{new} = V_i^{old} + \Delta V_i$$

比例调节的优点是响应速度快、实现简单，缺点是存在静差，当系统存在固有的水力失调时无法完全消除偏差。$K_p$的取值需要根据管网特性进行调试，$K_p$过大可能导致系统振荡，$K_p$过小则调节作用太弱。

#### 5.3.2 PI调节算法

PI调节在比例调节的基础上引入了积分环节，可以消除静差、提高控制精度。PI调节的控制律为：

$$u(t) = K_p \cdot e(t) + K_i \cdot \int_0^t e(\tau) d\tau$$

其中，$K_p$为比例系数，$K_i$为积分系数，$u(t)$为控制输出。在离散实现中，积分项采用累加近似：

$$\int_0^t e(\tau) d\tau \approx \sum_{k=0}^{n} e(k) \cdot \Delta t$$

PI调节的离散形式为：

$$\Delta V_i = K_p \cdot [e_i(n) - e_i(n-1)] + K_i \cdot e_i(n)$$

积分环节的作用是累积历史偏差，当存在静差时积分项会持续增长，直到偏差被消除。积分饱和是PI控制中常见的问题，当阀门开度达到限值后需要限制积分作用的继续增长，避免产生超调。

#### 5.3.3 模糊控制算法

模糊控制适用于难以建立精确数学模型、存在大滞后和强耦合的复杂控制系统。本系统实现的模糊控制器采用二维结构，以室温偏差$e$和偏差变化率$ec$作为输入，以阀门开度调整量$\Delta V$作为输出。

模糊控制器的设计包括三个步骤：模糊化、模糊推理、解模糊化。模糊化将精确的输入量转换为模糊语言变量，典型的语言值为{NB, NM, NS, ZO, PS, PM, PB}，分别表示负大、负中、负小、零、正小、正中、正大。隶属度函数可采用三角形、梯形或高斯形函数。

模糊推理基于一组if-then规则进行，常用的控制规则表如下：

| $e$ \ $ec$ | NB | NM | NS | ZO | PS | PM | PB |
|-----------|----|----|----|----|----|----|-----|
| NB        | PB | PB | PM | PM | PS | ZO | ZO |
| NM        | PB | PB | PM | PS | PS | ZO | NS |
| NS        | PM | PM | PM | PS | ZO | NS | NS |
| ZO        | PM | PS | PS | ZO | NS | NS | NM |
| PS        | PS | PS | ZO | NS | NS | NM | NM |
| PM        | PS | ZO | NS | NM | NM | NB | NB |
| PB        | ZO | ZO | NM | NM | NB | NB | NB |

解模糊化将模糊输出转换为精确的控制量，常用方法包括重心法、最大隶属度法等。重心法的计算公式为：

$$\Delta V = \frac{\sum_{i=1}^{n} \mu_i \cdot v_i}{\sum_{i=1}^{n} \mu_i}$$

其中，$\mu_i$为第$i$条规则输出模糊集合的隶属度，$v_i$为对应的代表值。

### 5.4 异常检测算法

异常检测是预防设备故障和保障系统安全的重要手段，本系统采用基于时序数据的异常检测算法，能够识别温度突变、压力异常、流量偏离等不正常状态。系统采用滑动窗口统计与机器学习相结合的方法，实现对异常状态的早期预警。

#### 5.4.1 统计方法异常检测

统计方法基于历史数据建立正常运行的统计模型，当实时数据偏离模型预测范围时判定为异常。本系统采用基于均值和标准差的3$\sigma$准则作为基础的异常检测方法。

设窗口长度为$W$，对于任意时间点$t$，计算前$W$个样本的均值$\mu$和标准差$\sigma$：

$$\mu = \frac{1}{W} \sum_{i=t-W}^{t-1} x_i$$

$$\sigma = \sqrt{\frac{1}{W} \sum_{i=t-W}^{t-1} (x_i - \mu)^2}$$

其中，$x_i$为第$i$时刻的采样值。若当前值$x_t$满足$\mu - 3\sigma \leq x_t \leq \mu + 3\sigma$，则判定为正常，否则判定为异常。

为适应运行工况的变化，采用指数加权移动平均更新均值和标准差：

$$\mu_t = \alpha \cdot x_t + (1-\alpha) \cdot \mu_{t-1}$$

$$\sigma_t^2 = \alpha \cdot (x_t - \mu_t)^2 + (1-\alpha) \cdot \sigma_{t-1}^2$$

其中，$\alpha$为遗忘因子，通常取0.1-0.3之间的值。

#### 5.4.2 机器学习方法异常检测

对于更复杂的异常模式，系统采用基于孤立森林（Isolation Forest）的机器学习方法进行异常检测。孤立森林的核心思想是异常点在特征空间中更容易被"孤立"，即通过随机划分特征空间，异常点通常只需要更少的划分次数就能被分离出来。

孤立森林的构建过程如下：从样本数据中随机选择一个特征和一个划分值，将样本空间递归划分为两个子空间，直到满足终止条件（树的深度达到限定值或子空间样本数少于最小值）。每棵树的划分过程相互独立，最终形成一片森林。

对于新样本$x$，计算其在各棵树中的平均划分深度：

$$s(x, n) = 2^{-\frac{E(h(x))}{c(n)}}$$

其中，$E(h(x))$为样本$x$在各棵树中划分深度的平均值，$c(n)$为样本数为$n$时的平均路径长度归一化因子。$s(x,n)$的取值范围为0到1，接近1表示样本为异常点的可能性更高。

本系统将孤立森林算法应用于温度、压力、流量等多维时序数据的异常检测，通过训练正常工况下的数据构建异常检测模型，运行时将实时数据输入模型计算异常分数，超过阈值的点标记为异常并产生告警。

#### 5.4.3 异常预测算法

在检测到异常后，系统进一步采用时间序列预测方法预测异常的发展趋势，为运维人员争取处理时间。本系统采用指数平滑法进行短期异常预测：

$$\hat{x}_{t+k} = \alpha \cdot x_t + \alpha(1-\alpha) \cdot x_{t-1} + \alpha(1-\alpha)^2 \cdot x_{t-2} + \cdots$$

其中，$\hat{x}_{t+k}$为$t+k$时刻的预测值，$\alpha$为平滑系数。指数加权移动平均给予近期数据更大的权重，能够较好地跟踪异常的变化趋势。

预测结果的置信区间采用Bootstrap方法估计，通过对历史残差进行重采样，计算预测值在不同置信水平下的分位数，为决策提供参考。

### 5.5 能耗优化算法

能耗优化旨在在满足供热需求的前提下最小化能源消耗和运行成本。本系统采用基于线性规划和启发式算法的混合优化方法，综合考虑设备特性、运行约束和能源价格等因素。

#### 5.5.1 优化模型构建

能耗优化问题可表述为以下数学规划模型：

**目标函数**：最小化总运行成本

$$\min \sum_{t=1}^{T} \sum_{i=1}^{I} (C_i^{fuel} \cdot Q_i(t) + C_i^{start} \cdot y_i(t) + C_i^{om} \cdot P_i(t))$$

其中，$T$为优化时段数，$I$为设备数量，$C_i^{fuel}$、$C_i^{start}$、$C_i^{om}$分别为设备$i$的燃料成本、启停成本、运维成本系数，$Q_i(t)$、$y_i(t)$、$P_i(t)$分别为设备$i$在时段$t$的出力、启停状态、用电功率。

**约束条件**包括：

热平衡约束：$\sum_{i=1}^{I} Q_i(t) = Q^{demand}(t) + Q^{loss}(t)$

设备出力约束：$Q_i^{min} \cdot y_i(t) \leq Q_i(t) \leq Q_i^{max} \cdot y_i(t)$

爬坡速率约束：$|Q_i(t) - Q_i(t-1)| \leq R_i^{ramp}$

最小运行时间约束：$y_i(t) - y_i(t-1) \leq y_i(t+1) + y_i(t+2) + \cdots + y_i(t+T_i^{on})$

最小停机时间约束：$y_i(t-1) - y_i(t) \leq y_i(t+1) + y_i(t+2) + \cdots + y_i(t+T_i^{off})$

#### 5.5.2 求解算法

上述优化问题是一个混合整数规划问题，本系统采用分支定界法结合线性规划松弛进行求解。分支定界法的基本思想是：先求解问题的线性规划松弛得到下界，然后通过分支操作枚举整数变量，通过定界操作剪枝不可行或非最优的分支，直至找到满足所有整数约束的最优解。

对于大规模问题，系统还实现了基于遗传算法的启发式求解方法。遗传算法的流程包括：初始化种群、计算适应度、选择操作、交叉操作、变异操作、精英保留。适应度函数取目标函数的倒数，值越大表示个体越优。选择操作采用轮盘赌算法，交叉操作采用单点交叉或多点交叉，变异操作随机改变某些基因的值。

遗传算法的参数设置如下：种群规模取20-50，交叉概率取0.7-0.9，变异概率取0.01-0.05，最大迭代次数取100-500。算法收敛后输出最优的设备调度方案，包括各设备的启停状态和出力计划。

## 6 接口设计

### 6.1 前端组件接口

前端组件采用Props接收父组件传递的数据和回调函数，组件内部通过emit触发事件向上传递状态变化。主要组件接口设计如下：

数据展示类组件接收数据源、显示格式、筛选条件等props，输出展示后的视图；控制类组件接收当前状态、控制参数、权限标识等props，输出用户的操作行为；图表类组件接收图表类型、数据源、样式配置等props，输出交互事件和导出操作。

组件间的通信通过Pinia状态管理进行，共享数据存储在对应的Store中，各组件通过访问Store获取和修改状态，确保数据的一致性和可追溯性。

### 6.2 API接口规范

后端API采用RESTful风格设计，遵循统一的接口规范。接口路径以资源名称命名，使用HTTP方法表示操作类型：GET用于查询、POST用于创建、PUT用于更新、DELETE用于删除。

接口响应采用统一的JSON格式，包含code（状态码）、message（提示信息）、data（业务数据）三个字段。分页查询接口的data字段包含list（数据列表）、total（总记录数）、pageNum（当前页码）、pageSize（每页条数）四个子字段。

接口安全采用Token认证机制，用户登录成功后获取Token，后续请求在请求头中携带Token进行身份验证。敏感操作需要额外的管理员权限校验。

## 7 数据模型

### 7.1 核心实体

系统核心实体包括热用户（HeatUser）、建筑物（Building）、换热站（Station）、锅炉（Boiler）、管网节点（Node）、管网管段（Pipe）、告警记录（Alarm）等。每个实体包含基本属性和业务属性，基本属性为所有实体共有的标识、名称、状态、创建时间、更新时间等字段，业务属性根据实体类型定义。

实体之间的关系为：换热站与建筑物为一对多关系，建筑物与热用户为一对多关系，换热站与锅炉为多对多关系（通过一网连接），管网节点与管网管段为图拓扑关系。

### 7.2 时序数据

时序数据采用时序数据库存储，按时间线组织数据，支撑高效的时间范围查询和聚合计算。主要的时序数据包括温度测点数据、压力测点数据、流量测点数据、阀门开度数据、设备状态数据等。

时序数据的存储采用压缩策略，根据数据的重要性分级存储：高频详细数据保留7天，低频汇总数据保留90天，统计数据保留1年。超过保留期限的数据自动归档或清理。

## 8 安全性设计

### 8.1 权限控制

系统采用基于角色的访问控制（RBAC）模型，将权限划分为功能权限和数据权限两个维度。功能权限控制用户可访问的菜单和操作按钮，数据权限控制用户可查看的数据范围（如所属区域的换热站数据）。

系统预定义管理员、调度员、运维员、访客四种角色，每种角色关联不同的权限集合。用户可以被分配一个或多个角色，拥有的权限为所有角色权限的并集。

### 8.2 操作审计

系统对所有敏感操作进行审计记录，包括用户登录登出、数据增删改、系统参数修改、控制指令下发等操作。审计日志包含操作时间、操作人员、操作类型、操作内容、操作结果、来源IP等字段。

审计日志采用追加写入的方式存储，不可修改和删除，确保操作行为的可追溯性。日志保留期限不少于180天，满足合规要求。

## 9 性能要求

### 9.1 响应时间

系统各功能页面的加载时间不超过3秒，数据查询响应时间不超过1秒，图表渲染时间不超过500毫秒。实时数据刷新间隔为30秒，可配置调整。

### 9.2 并发能力

系统支持500个以上用户同时在线，支持100个以上并发请求。核心业务的处理吞吐量为每秒1000条以上数据记录。

### 9.3 可用性

系统整体可用性不低于99.9%，即年度非计划停机时间不超过8.76小时。系统支持7×24小时连续运行，关键数据每日自动备份。
